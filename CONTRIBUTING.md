# Contributing to AGENTS.md

This project is a rules document backed by empirical evidence. Contributions should strengthen one or the other.

## What's welcome

**Propose a new rule.** Open an issue with:
- The rule, phrased as an imperative.
- The failure mode it prevents.
- The experiment or observation that motivates it.
- Why it generalizes beyond a single project.

**Add supporting evidence.** If you've run your own multi-agent experiments and the data speaks to one of the existing rules — supporting or contradicting — open an issue or PR with your methodology and numbers.

**Improve clarity.** Rules should be readable in one sitting. Typo fixes, sentence-level clarifications, better examples: all welcome.

## What's not welcome

- Rule proposals based on intuition or single-run demos. Bring data.
- Rules that only apply to a specific framework, vendor, or toolchain. The rules should generalize.
- Philosophical debates about agent autonomy. Keep it operational.

## Process

1. **For small changes** (typo, clarity): open a PR directly.
2. **For new rules or methodology changes:** open an issue first, let's discuss before you spend effort on a PR.
3. **For contradicting evidence:** always open an issue. If your data overturns a rule, we might need to rewrite more than one.

## Attribution

Contributors will be acknowledged in a future `CONTRIBUTORS.md` once there are a few. For substantial contributions (a new rule with supporting evidence), co-authorship on derivative publications is negotiable.
